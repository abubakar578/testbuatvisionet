Story behind the features

Odoo behavior : For service product, the invoice control can be done only on ordered quantity for 100%. It's not possible to plan the invoice based on a milestone validation and not easy to simply invoice an order line in %.

CLuedoo behavior : By adding the notion of invoice term, each sales order line can have the list of milestone and the milestone validation will automatically generate the related invoice.

Business Case + : In some sales cases, we usually have payment terms split into several payment, such as Payment I : 30%, Payment II : 50%, Payment III : 20%. How to make it? How can we know how much %, we already invoiced? The answer is Invoice milestone. We can split the invoice, schedule it, and control the % invoiced.
